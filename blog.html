Key Fundamentals of Backend Development

What is Backend?

The backend refers to the part of a software application or website that runs on the server side, handling the logic, database interactions, authentication, and data processing. It essentially powers the front end, which is what users see and interact with.

Backend roadmap:

https://roadmap.sh/backend

Authorization

Ensures that users have permission to access certain resources or perform actions based on their roles and permissions.

Authentication

Verifies the identity of a user or system, typically using credentials like usernames and passwords.This process ensures that only authorized individuals or systems can access specific resource and perform specific actions.

Validation

Ensures that the data being processed is correct, complete, and in the expected format. This helps in preventing errors and maintaining data integrity.

Error Handling

Manages and responds to errors or exceptions that occur during processing, ensuring the system remains stable and provides meaningful feedback to users.

Data Persistence

Involves storing and retrieving data from databases or other storage systems. This includes CRUD (Create, Read, Update, Delete) operations.

Security

Protects the system and data from unauthorized access, attacks, and vulnerabilities. This includes practices like encryption, input sanitization, and secure coding.

Scalability

Ensures that the system can handle increased load or traffic without performance degradation, often through techniques like load balancing and horizontal scaling.

Performance Optimization

Enhances the efficiency and speed of the system, including optimizing queries, caching responses, and minimizing resource usage. This ensures the application can handle more requests with lower latency, providing a better overall user experience.

API Design

Involves creating and managing application programming interfaces (APIs) that allow different software components to communicate and interact with each other.

Logging

Records system activities and events for monitoring, debugging, and auditing purposes. This helps in tracking down issues and understanding system behavior.

Testing

Ensures the reliability and functionality of the backend through various types of tests, such as unit tests, integration tests, and end-to-end tests.

Concurrency Management

Handles multiple processes or threads running simultaneously, ensuring that resources are managed efficiently and consistently.

JWT

JSON Web Token

A JSON Web Token (JWT) is like a secret message you send to a friend with a special code that only you and your friend can understand. It ensures that the message is secure and hasn’t been tampered with. Companies use JWTs in a similar way to verify your identity and protect your data online.

Security Considerations:
Signature Stripping, Cross-Site Request Forgery (CSRF), Cross-Site Scripting (XSS)

How JWT Authentication Works?

1. User Login

When a user attempts to log in, the server authenticates the user's credentials (typically username and password). Upon successful authentication, the server generates a JWT containing the user's information and signs it with a secret key.

                            
                            const jwt = require('jsonwebtoken');
                            const token = jwt.sign({ userId: user.id }, 'your-256-bit-secret', { expiresIn: '1h' });

2. Client Receives JWT

The server sends the generated JWT back to the client. The client stores this token, typically in local storage or a cookie.

                            
                                localStorage.setItem('token', token);

3. Client Makes Authenticated Requests

For subsequent requests, the client includes the JWT in the Authorization header.

GET /protected-resource
Authorization: Bearer

4. Server Verifies JWT

Upon receiving a request with a JWT, the server verifies the token’s signature to ensure its integrity and authenticity.

                            
                                const token = req.headers.authorization.split(' ')[1];
                                jwt.verify(token, 'your-256-bit-secret', (err, decoded) => {
                                if (err) {
                                    return res.status(401).send('Unauthorized');
                                }
                                req.user = decoded;
                                next();
                                });

JWTs are widely used for authentication because they allow you to verify the identity of a user without needing to store session data on the server. When a user logs in, the server generates a JWT and sends it to the client. The client includes this token in the Authorization header of subsequent requests. The server can then verify the token to authenticate the user.
